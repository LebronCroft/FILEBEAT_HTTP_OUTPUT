# Security Policy

## Reporting a Vulnerability

Please do not report security vulnerabilities in public issues.

Send a private report to the project maintainer with:

- A description of the issue.
- Affected versions or commits.
- Reproduction steps or proof of concept, if available.
- Suggested mitigation, if known.

The maintainer should acknowledge the report, investigate the impact, and coordinate a fix before public disclosure.

## Handling Sensitive Data

Do not include real tokens, passwords, private keys, certificates, internal hostnames, internal paths, or production endpoint details in issues, pull requests, examples, or tests.

Use placeholders such as:

- `example.com`
- `localhost`
- `127.0.0.1`
- `your-token-here`
- `your-cert.pem`
- `your-key.pem`
